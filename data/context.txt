Lineamientos técnicos
- Para reactividad usar React.
- Gestor de paquetes: Yarn.
- Gestor de estado (si es necesario): Zustand.
- Estilos: Tailwind CSS.
- Las páginas de Next deben ser server por defecto.
- Usar componentes y hooks para encapsular lógica.
- La aplicación debe ser responsive.
- Los colores oficiales están en data/colores.md.
- Para consumir APIs, seguir mejores prácticas y una estructura modular.
- Documentar el código en español, pero usar variables en ingles.

Estado actual del frontend
- Login/registro siempre redirige a `/my-profile`.
- Existe guardia de orden obligatorio: no permite saltar pasos por URL, salvo `/my-profile`.
- `/my-profile` muestra línea de tiempo y botón `Comenzar/Continuar pre-inscripción`.
- `/pre-inscription-affidavit` muestra declaración inicial HTML desde `system_documents` y registra aceptación.
- `/payment` muestra obligaciones generadas después de aprobación administrativa de declaración jurada.
- `/resume` muestra resumen real, checkbox de conformidad y confirma datos.
- `/satisfaction` registra encuesta final luego de confirmar datos.
- El home puede tener diseño propio (restaurado por el usuario) sin afectar el resto de pasos.
- Los pasos del flujo usan navegación básica con botones `Regresar` y `Siguiente`.
- En `registration-form` el botón `Siguiente` queda deshabilitado por ser el último paso.
- El layout privado incluye:
  - Header con logo UNI (`public/uni.png`), texto "Inscripciones 2026-1" y botón "Mi perfil".
  - Footer sutil y poco cargado.
- La ruta `/my-profile` está activa para el acceso desde el header.
- La ruta `/payments` se considera legacy; el flujo principal usa `/payment`.
- La campana lista pendientes, pero el flujo debe seguir el orden obligatorio.

Estados y validaciones por paso
- `personal-data` requiere declaración inicial aceptada.
- `sworn-affidavit` solo completa cuando la DDJJ está aprobada por administración.
- `payment` se habilita cuando se generan obligaciones; `resume` requiere todos los pagos validados.
- `satisfaction` va después de `data_confirmed=true`.

Contratos de API
- Base en `src/lib/applicant.ts`.
- Nuevos endpoints: `preinscription-consent`, `payments`, `confirm-data`, `satisfaction-survey`.
